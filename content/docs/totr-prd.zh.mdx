## 6. 关键功能 & 验收标准（MVP）

### 6.1 解释页（/meme/totr）

- 模块：60s 概览、起源时间线、视觉元素、拼写对照、与 ПЫ 的关系、FAQ。
- 媒体：仅官方 Embed（TikTok/YouTube/IG），不抓取源视频。
- 结构化数据：`FAQPage`、`Article`、侧边 `ItemList`（热门梗导航）。
- SEO：标题 ≤60 字符；元描述 140–160；OG/Twitter；语义化 H1/H2；内链到生成器/下载/对比。
- 指标：移动 Lighthouse ≥90；LCP ≤2.5s（p75）；FAQ Rich Result 可见。

### 6.2 生成器（/generator/totr）

- 画布：Fabric.js；自绘“海绵体”风格 SVG/WebP 底图（非官方形象）。
- 功能：文本/贴纸（自有）/表情、缩放旋转、PNG 导出（默认带水印）。
- 限制：不内置受版权素材；上传需勾选“我有权利使用”；MIME 白名单、≤10MB。
- 引导：3 个“一键样式”模板；示例文案（EN/RU）。
- 指标：3 步导出；导出 P95 < 2s；失败率 < 1%；移动端兼容。

### 6.3 模板下载（/download/totr-template）

- 内容：≥3 张自绘底图 + 在线编辑链接（Canva/Figma）；授权条款清晰。
- 变现：正文信息流 + “解锁更多模板”支付入口。

### 6.4 对比页（/compare/totr-vs-py）

- 模块：概念差异、视觉对照、投票（本地存储防刷）。
- 目标：提升停留与回访，增强站内链接权重。

### 6.5 热度追踪（/trends/totr）

- 内容：出圈节点 + 爆款链接索引（仅外链/Embed），每日 1–3 条更新。
- 目标：形成“新闻式新鲜度信号”，带动长尾关键词。

### 6.6 合规（/policy/ugc + 全站）

- 仅自绘戏仿体 + 官方 Embed；UGC 勾选权利声明；DMCA/举报表单（邮箱、链接、理由）。
- SLA：24h 响应；必要时 410 移除；TOS/隐私明示工具与资讯性质。

- 前端：Next.js（App Router，ISR）+ Tailwind；Fabric.js（画布）。
- 部署：Vercel（Edge/ISR）。
- 存储/CDN：Cloudflare R2 + CDN；图片压缩与缓存策略。

- 观测：Vercel Analytics + GA4 + Amplitude；错误上报（Sentry 可选）。

## 17. 风险 & 预案

- 版权投诉：仅自绘 + 官方 Embed + DMCA 24h；侵权内容 410。
- 热度衰减：模板化复刻至新梗；“热门梗导航”承接流量。
- 广告 RPM 低：提高信息密度与内链；A/B 广告位；生成器闭环拉高每访 PV。
- 支付转化低：凸显去水印/4K/字体价值；成功页加二段激励；模板风格差异化。
